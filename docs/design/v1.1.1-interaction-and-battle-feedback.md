# 《雾下余粮》V1.1.1：交互与战斗表现优化

> 文档状态：V1.1.1 开发基线  
> 依赖版本：V1.1 格子探索与回合制战斗  
> 变更性质：增强危险反馈、交互提示和战斗表现，不改变 20×20 格子探索的核心循环

## 1. 调整目标

V1.1 已实现三态战争迷雾、明雷敌人、接敌、独立回合制战斗、切割和撤离。V1.1.1 解决玩家无法清晰理解“敌人何时发现我”“为什么突然进入战斗”“现在能做什么”的问题。

统一反馈原则：

1. 视觉：高亮、图标、颜色和动效。
2. 文字：说明危险来源或当前可执行操作。
3. 声音：接口预留，本版本不要求音频资源。

## 2. 敌人感知状态机

统一状态调整为：

```text
Idle / Wander → Alert → Chase → AttackIntent → Battle
                         ↓             ↓
                       Return       Cooldown
```

- `Idle / Wander`：正常待机或游荡，无特殊提示。
- `Alert`：玩家进入感知范围，敌人头顶显示 `!`，持续 `alertDuration` 个地图回合。
- `Chase`：确认玩家位置，敌人使用红色追踪标记并向玩家靠近。
- `AttackIntent`：敌人下一次行动可进入玩家格，显示强危险提示；下一地图回合仍满足条件时接敌。
- `Return`：超出追踪或领地限制后返回出生区域。
- `Cooldown`：逃跑成功后暂时脱战。
- `Dead`：移出 AI 列表并留下尸体。

提示文案基线：

- Alert：`附近的怪物似乎察觉到了你的存在。`
- Chase：`危险！某个未知生物正在接近。`
- AttackIntent：`它盯上你了。下一回合将会接敌。`

攻击意图属于预测信息，只展示敌人下一行动目标，不直接造成地图伤害。

## 3. 战斗进入与结束

接敌流程：地图冻结 → 敌人名称特写 → 画面拉近/淡入 → 战斗 UI。

战斗结束不再直接返回地图：

- 胜利：显示“击败敌人”“尸体已留下，可以切割”，再回原地图。
- 失败：显示死亡结果，再返回庇护所。
- 逃跑：显示脱战结果，再回原地图；敌人进入 `Cooldown`。

转场由 `battleTransition` 控制，结果停留时间由 `battleResultDelay` 控制。声音和复杂角色动画列为后续增强。

## 4. 速度与行动顺序

玩家和怪物新增 `speed`。`useSpeedOrder = true` 时，战斗开始按速度决定先手：

1. 速度高者先行动。
2. 玩家先手时，玩家选择行动，随后敌人行动。
3. 敌人先手时，敌人先行动，再开放玩家按钮；之后每轮保持相同先后顺序。
4. 速度相同默认玩家先手，后续可增加随机或接敌方优先规则。

战斗 UI 必须显示双方速度，并在玩家可操作时显示“轮到你行动”。玩家行动区整体高亮，默认攻击按钮提供轻微呼吸效果。敌人行动期间按钮不可用，并显示“敌人正在行动”。

## 5. 地图交互提示

玩家站在可交互格时，侧栏主交互按钮必须切换为具体动作：

- 尸体格：`切割尸体`，危险色高亮；背包满时显示 `背包已满` 并禁用。
- 撤离点：`开始撤离`，金色高亮。
- 普通格：`暂无交互`，禁用。
- 种植点：结构预留，未来显示 `种植` 或 `收获`。

切割仍按地图回合推进，并显示开始、所需回合和最终获得数量。敌人在切割期间可以继续行动。

## 6. 配置增量

```json
{
  "player": { "speed": 10 },
  "monster": {
    "speed": 8,
    "alertDuration": 2,
    "detectRadius": 3,
    "attackIntentRange": 1
  },
  "battle": {
    "useSpeedOrder": true,
    "battleTransition": true,
    "battleResultDelay": 2
  },
  "ui": {
    "showEnemyAlert": true,
    "showAttackIntent": true,
    "highlightInteract": true
  }
}
```

`detectRange` 作为 V1.1 兼容字段暂时保留；运行时优先读取 `detectRadius`。

## 7. 实现优先级

### P0

- Alert、Chase、AttackIntent 状态及文字/地图提示。
- 敌人下一回合接敌的攻击意图预告。
- 战斗进入和结束转场。
- 玩家、怪物速度配置及速度先后手。
- 玩家行动区高亮与回合提示。
- 切割和撤离按钮的上下文提示与高亮。

### P1

- 更完整的敌人特写、胜利演出、攻击方向箭头和危险区域。

### P2

- 音效、屏幕震动、高疯狂特殊反馈和敌人专属警觉动作。

## 8. 验收标准

- 玩家首次进入敌人感知范围时，不会在无提示的情况下立即接敌。
- 敌人从 Alert 进入 Chase 时有明确文字反馈。
- 敌人相邻且下次会接敌时，地图与 UI 同时显示攻击意图。
- 战斗前后均存在独立过渡画面。
- 更高速度的一方在第一轮先行动，战斗 UI 显示双方速度。
- 玩家可行动时按钮明显高亮；不可行动时不能重复点击。
- 站在尸体或撤离点时，主交互按钮不需要玩家猜测用途。
