# 《雾下余粮》V1.3 地图内容生产与巢穴系统

> 版本名：V1.3 Map Authoring and Nest System  
> 基线：`main` 与 `docs/design/current-game-status.md`  
> 本版本不修改庇护所、生命、饥饿及 Demo 胜负规则。

## 1. 版本目标

- 地图宽高由配置控制，支持 10～50 格。
- 配置页提供轻量可视化地图编辑器。
- 固定怪物点之外，支持带 Seed 的随机怪物布点。
- 巢穴仍是统一怪物实体，只通过 `spawnConfig` 获得产出能力。
- 当前外出的 Seed、怪物和巢穴进度可随存档恢复。
- 保持旧配置、旧存档和默认 20×20 流程兼容。

## 2. 地图与视口

地图配置增加或规范以下字段：

```ts
interface MapConfig {
  id: string;
  name: string;
  width: number;
  height: number;
  playerSpawn: GridPosition;
  extractionPoints: ExtractionPointConfig[];
  obstacles: GridPosition[];
  monsterSpawns: MonsterPlacementConfig[];
  random: MapRandomConfig;
  randomSpawnRules: RandomSpawnRuleConfig[];
}
```

宽高必须是 10～50 的整数。运行时最多显示玩家附近 20×20 格，不通过继续缩小格子展示完整大地图；视口随玩家移动。配置编辑器使用可滚动画布。

地图缩小时，越界障碍、固定怪物点和撤离点会被清理；玩家出生点会被约束在边界内。地图必须有且仅有一个玩家出生点，并至少有一个撤离点。

## 3. 地图编辑器

配置页新增“地图编辑”分类，提供：

- 地面、障碍、玩家出生点、撤离点、固定怪物和删除工具。
- 地图宽高修改及越界数据清理。
- 点击格子放置对象，地面和障碍支持连续绘制的数据入口。
- 清空地图对象。
- 固定 Seed 开关、生成 Seed 和复制 Seed。
- 使用未保存草稿试玩当前地图，退出后返回编辑器且不污染正式存档。
- 完整配置继续沿用全局 JSON 导入、导出、校验和保存。

第一版编辑器不是商业级关卡工具，不包含图层、多选、框选和实时热更新。撤销、重做可在后续增加。

## 4. 随机怪物布点

```ts
interface RandomSpawnRuleConfig {
  id: string;
  enabled: boolean;
  monsterConfigId: string;
  minCount: number;
  maxCount: number;
  allowedArea?: RectAreaConfig;
  excludedAreas?: RectAreaConfig[];
  minDistanceFromPlayerSpawn: number;
  minDistanceFromExtraction: number;
  minDistanceBetweenSameType: number;
  minDistanceBetweenAnyMonster: number;
  placementAttempts: number;
}
```

初始化顺序固定为：地形 → 玩家出生点 → 撤离点 → 固定怪物 → 随机怪物 → 战争迷雾。

随机布点不能占用障碍、出生点、撤离点或其他怪物，必须满足配置距离。达到最大尝试次数后保留已成功放置的结果，不阻塞启动、不无限循环。

## 5. Seed 规则

```ts
interface MapRandomConfig {
  useFixedSeed: boolean;
  seed: string;
}
```

- 固定 Seed：相同地图配置和 Seed 得到相同随机点位。
- 非固定 Seed：新外出生成新 Seed。
- 当前外出的 Seed 写入存档；刷新后沿用该 Seed 和运行时快照。
- 最终布点使用项目内的可复现伪随机生成器，不直接依赖 `Math.random()`。

## 6. 巢穴型怪物

巢穴不新增 `NestEntity`。它继续使用怪物的生命、防御、速度、战斗、死亡、尸体和切割流程，仅增加：

```ts
interface MonsterSpawnConfig {
  enabled: boolean;
  monsterConfigId: string;
  intervalTurns: number;
  initialDelayTurns: number;
  maxAliveChildren: number;
  maxTotalChildren?: number;
  spawnRadiusMin: number;
  spawnRadiusMax: number;
  spawnOnVisibleTile: boolean;
  requireWalkableTile: boolean;
  childHomeLinkedToSpawner: boolean;
}
```

默认巢穴不移动、不感知、不追踪、不主动接敌、攻击为 0。玩家主动进入巢穴格时进入正常战斗；巢穴回合只显示蠕动，不造成伤害。

每个地图回合，存活巢穴更新倒计时。到期后检查存活子怪上限、总产出上限和合法位置，再创建配置指定的普通怪物。合法位置必须可通行且不能占用玩家、怪物、巢穴或撤离点。

子怪记录 `spawnedByMonsterId`。开启 `childHomeLinkedToSpawner` 时，以巢穴位置作为 Home 中心。巢穴死亡后停止产出，已经生成的子怪继续存在。

巢穴使用独立地图轮廓和提示：产出时提示“巢穴中爬出了新的生物”，破坏时提示“巢穴停止了蠕动”。死亡后仍沿用普通尸体和切割系统，只产出异变肉。

## 7. 配置和存档兼容

旧配置迁移时自动补充 `extractionPoints`、`random` 和 `randomSpawnRules`，不改变旧地图固定点位。新巢穴作为内置怪物配置存在。

当前外出快照保存地图 ID、Seed、地图回合、玩家本局状态、已生成怪物、子怪归属、巢穴倒计时与总产出、尸体、已访问格和迷雾记忆。正常撤离或失败时清除外出快照。

## 8. 怪物配置管理

配置页允许新增怪物、复制怪物，以及删除无引用的非内置怪物。内置怪物不可删除。删除前检查固定点、随机规则和巢穴产出引用；存在引用时拒绝删除。

`spawnConfig` 继续由通用嵌套配置表单编辑，不建立独立巢穴页面。

## 9. 验收与测试

自动测试覆盖 40×40、50×50 尺寸，地图缩小越界清理，固定 Seed 复现，随机点位避让和空间不足退出，以及巢穴间隔、存活上限、子怪死亡补充、巢穴死亡停止产出等规则。

## 10. 明确不做

- 完整程序化地图、多层地图及地图切换。
- Boss 专用系统、巢穴升级、巢穴联动或巢穴主动攻击。
- 装备掉落、制作、怪物阵营和繁殖模拟。
- 动态地形破坏、云端地图分享、在线 Seed 排行榜。
- 完整商业级关卡编辑器。
