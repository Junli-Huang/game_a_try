# 《雾下余粮》V1.3 地图内容生产与巢穴系统

> 版本名：V1.3 Map Authoring and Nest System
>
> 实现状态：`feat/v1.3-completion` 分支实际代码、自动测试与生产构建
> 本版本不修改庇护所、生命、饥饿及 Demo 胜负规则。

## 1. 版本目标

- 地图宽高由配置控制，支持 10～50 格。
- 配置页提供轻量可视化地图编辑器。
- 固定怪物点之外，支持带 Seed 的随机怪物布点。
- 巢穴仍是统一怪物实体，只通过 `spawnConfig` 获得产出能力。
- 当前外出的 Seed、怪物和巢穴进度可随存档恢复。
- 保持旧配置、旧存档和默认 20×20 流程兼容。

## 2. 地图与视口

地图配置增加或规范以下字段：

```ts
interface MapConfig {
  id: string;
  name: string;
  width: number;
  height: number;
  playerSpawn: GridPosition;
  extractionPoints: ExtractionPointConfig[];
  obstacles: GridPosition[];
  monsterSpawns: MonsterPlacementConfig[];
  random: MapRandomConfig;
  randomSpawnRules: RandomSpawnRuleConfig[];
}
```

宽高必须是 10～50 的整数。运行时最多显示玩家附近 20×20 格，不通过继续缩小格子展示完整大地图；视口随玩家移动。配置编辑器使用可滚动画布。

地图缩小时，编辑器会清理越界障碍、固定怪物点和撤离点，并把玩家出生点约束在边界内；如果撤离点被全部清理，会补回一个默认撤离点。尺寸调整后仍需通过配置校验，校验会拒绝出生点、撤离点、障碍和固定怪物之间的非法重叠。地图数据以单个 `playerSpawn` 表示一个出生点，配置校验要求至少一个撤离点。

## 3. 地图编辑器

配置页新增“地图编辑”分类，提供：

- 选择、地面、障碍、玩家出生点、撤离点、固定怪物、矩形随机怪物区域和删除工具。
- 地图宽高修改及越界数据清理。
- 点击格子放置对象；地面、障碍和删除工具支持按住拖动连续绘制。选择工具可打开对象属性面板，修改坐标、固定怪物数量/类型、撤离回合，或随机区域的数量、范围、距离和尝试次数。
- 矩形随机怪物区域通过依次点击两个角创建，并在编辑器中显示范围叠层。
- 清空障碍、固定怪物和随机区域；为保持地图可启动，不清除玩家出生点和撤离点。
- 每张草稿地图最多保留 50 步撤销/重做历史；支持单张地图 JSON 导入、导出和仅恢复默认地图。
- 固定 Seed 开关、生成 Seed 和复制 Seed。
- 使用未保存草稿试玩当前地图，退出后返回编辑器且不污染正式存档。
- 完整配置继续沿用全局 JSON 导入、导出、校验和保存。

第一版编辑器不是商业级关卡工具，不包含图层、多对象选择、对象框选或运行中完整热更新。

## 4. 随机怪物布点

```ts
interface RandomSpawnRuleConfig {
  id: string;
  enabled: boolean;
  monsterConfigId: string;
  minCount: number;
  maxCount: number;
  allowedArea?: RectAreaConfig;
  excludedAreas?: RectAreaConfig[];
  minDistanceFromPlayerSpawn: number;
  minDistanceFromExtraction: number;
  minDistanceBetweenSameType: number;
  minDistanceBetweenAnyMonster: number;
  placementAttempts: number;
}
```

初始化顺序固定为：地形 → 玩家出生点 → 撤离点 → 固定怪物 → 随机怪物 → 战争迷雾。

随机布点会避开障碍、出生点、撤离点、固定怪物的配置坐标和此前成功的随机点位；同时检查配置的出生点/撤离点距离，以及相对于固定点和已成功随机点的“任意怪物”与“同类型”距离。达到最大尝试次数后保留已成功放置的结果，不阻塞启动、不无限循环。

## 5. Seed 规则

```ts
interface MapRandomConfig {
  useFixedSeed: boolean;
  seed: string;
}
```

- 固定 Seed：相同地图配置和 Seed 得到相同随机点位。
- 非固定 Seed：新外出生成新 Seed。
- 户外探索状态会将当前 Seed、伪随机生成器状态与运行时快照写入存档；恢复后直接采用快照中的怪物、尸体、巢穴进度、已发现巢穴和迷雾记忆，不重新生成现有随机点位，也不会改变后续依赖运行时随机数的结果。
- 最终布点使用项目内的可复现伪随机生成器，不直接依赖 `Math.random()`。

精确恢复涵盖已持久化的户外探索状态和地图事件服务的已触发事件、计数与间隔；战斗、战斗转场和事件选择框本身不在快照中。若在这些界面中刷新，游戏会回到最近一次已保存的户外探索状态，而不是恢复界面中间态。

## 6. 巢穴型怪物

巢穴不新增 `NestEntity`。它继续使用怪物的生命、防御、速度、战斗、死亡、尸体和切割流程，仅增加：

```ts
interface MonsterSpawnConfig {
  enabled: boolean;
  monsterConfigId: string;
  intervalTurns: number;
  initialDelayTurns: number;
  maxAliveChildren: number;
  maxTotalChildren?: number;
  spawnRadiusMin: number;
  spawnRadiusMax: number;
  spawnOnVisibleTile: boolean;
  requireWalkableTile: boolean;
  childHomeLinkedToSpawner: boolean;
}
```

默认巢穴不移动、不感知、不追踪、不主动接敌、攻击为 0。玩家主动进入巢穴格时进入正常战斗；巢穴回合只显示蠕动，不造成伤害。

每个地图回合，在普通怪物更新后，存活巢穴更新倒计时。到期后先重置下一次间隔，再检查存活子怪上限、总产出上限和合法位置；满足时创建配置指定的普通怪物。合法位置必须可通行且不能占用玩家、怪物、巢穴或撤离点。当前实现始终要求可通行格，`requireWalkableTile` 虽保留在配置中但不会改变这个行为。

子怪记录 `spawnedByMonsterId`。开启 `childHomeLinkedToSpawner` 时，以巢穴位置作为 Home 中心。巢穴死亡后停止产出，已经生成的子怪继续存在。

巢穴使用独立地图轮廓和提示：首次进入视野时提示“发现……仍在蠕动”，产出时可提示“巢穴中爬出了新的生物”，破坏时提示“巢穴停止了蠕动”。首次发现提示不受 UI 开关抑制；产出提示受“敌人警觉提示”UI 开关控制。死亡后仍沿用普通尸体和切割系统，只产出异变肉。

## 7. 配置和存档兼容

旧配置迁移时自动补充 `extractionPoints`、`random` 和 `randomSpawnRules`，不改变旧地图固定点位；如果旧怪物列表中缺少内置巢穴，则补入该内置配置。

户外探索快照保存地图 ID、Seed、伪随机生成器状态、地图回合、玩家本局状态、已生成怪物、子怪归属、巢穴倒计时与总产出、尸体、已访问格、已发现巢穴、地图事件服务状态和迷雾记忆。正常撤离或失败时清除外出快照；战斗和事件界面中间态不属于该快照。

## 8. 怪物配置管理

配置页允许新增怪物、复制任一怪物，以及删除无引用的任一非内置怪物。内置怪物不可删除。删除前检查固定点、随机规则和巢穴产出引用；存在引用时拒绝删除。还可恢复内置怪物配置；如自定义怪物仍被默认配置引用，恢复操作会拒绝执行。

`spawnConfig` 继续由通用嵌套配置表单编辑，不建立独立巢穴页面。

## 9. 验收与测试

现有自动测试覆盖：可复现伪随机序列及从保存状态续接、同 Seed 随机布点稳定性、避开障碍/出生点、空间不足时退出、40×40 和 50×50 尺寸校验、缩小地图时越界障碍与固定怪物清理、地图对象重叠校验、旧配置/旧外出快照迁移，以及巢穴首次发现、间隔、存活与总产出上限、无合法格跳过、子怪死亡后补充、巢穴死亡后停止产出。还覆盖外出快照对伪随机进度、巢穴倒计时/总产出及地图事件状态的精确恢复。测试没有覆盖地图编辑器的 DOM 交互、固定怪物距离约束、`requireWalkableTile` 开关差异，或在战斗/事件界面中刷新后的恢复行为。

## 10. 明确不做

- 完整程序化地图、多层地图及地图切换。
- Boss 专用系统、巢穴升级、巢穴联动或巢穴主动攻击。
- 装备掉落、制作、怪物阵营和繁殖模拟。
- 动态地形破坏、云端地图分享、在线 Seed 排行榜。
- 完整商业级关卡编辑器。
