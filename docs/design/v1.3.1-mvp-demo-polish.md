# 《雾下余粮》V1.3.1 MVP Demo 打磨

## 1. 版本目标

V1.3.1 不扩展大型玩法系统，重点改善首次理解、风险选择、结算反馈、默认节奏和移动端可靠性。目标是让新玩家在一分钟内理解 Demo 目标，在十至十五分钟内完成一次外出，并理解“异变肉维持生存但提高疯狂”的核心冲突。

默认胜利条件仍为成功撤离 3 次并累计带回 12 块异变肉；失败条件仍为外出失败达到 3 次，或庇护所的储备粮与异变肉同时耗尽。

## 2. 分段引导

引导使用现有模态框风格，一次只显示一条，关闭前暂停游戏输入。状态写入本地存档，旧存档缺少字段时自动补全；玩家可以跳过全部引导，并可在配置页重置。

| ID | 首次触发 | 说明重点 |
|---|---|---|
| `demo_goal` | 进入庇护所 | 胜负目标、异变肉与疯狂 |
| `outdoor_movement` | 进入户外 | 键盘、点击和触屏移动 |
| `fog_of_war` | 实际移动 | 可见、记忆和未探索区域 |
| `enemy_alert` | 敌人警觉/追踪/接敌意图 | 危险反馈与选择 |
| `first_battle` | 进入战斗 | 速度先手与四类行动 |
| `first_harvest` | 首次战斗胜利 | 尸体切割与地图回合 |
| `monster_meat` | 首次获得异变肉 | 饥饿、疯狂与攻击收益 |
| `first_nest` | 巢穴首次进入视野 | 产怪、被动接敌和破坏方式 |
| `first_extraction` | 首次站上撤离点 | 撤离回合与资源带回 |

## 3. 默认体验节奏

默认地图保持 20×20 和固定 Seed，增加第二撤离点。出生点附近保持温和，中部提供游荡者与安全撤离选择，远端巢穴与寻迹者形成高风险区域。系统配置范围仍为 10×10～50×50。

体验目标：30～90 秒出现首次有效信息，1～3 分钟首次战斗，3～5 分钟首次获得异变肉，10～15 分钟完成一次外出。具体时长必须结合手工试玩记录判断，不能仅由代码断言。

## 4. 结算统计

单次外出展示探索回合、探索格数、击杀、破坏巢穴、获得与食用异变肉、疯狂变化。失败至少区分战斗、饥饿和其他原因。最终胜利展示总外出、撤离、失败、击杀、巢穴破坏、带回肉和最高疯狂。

## 5. 移动端验收

检查 390×844、412×915、1366×768 和 1920×1080。移动端必须可以移动、战斗、使用二级菜单、切割、撤离和滚动结算；方向键不得触发页面误滚动。地图编辑器只要求可查看、基础点击、访问属性与退出。

## 6. 明确不做

不做 Boss、多地图、程序化地形、新装备或掉落、制作交易、多类型战利品、技能职业、剧情章节、NPC、服务端、联机、云存档、排行榜、完整 BGM、商业级编辑器、图层/多选/框选，以及生命和饥饿跨外出持久化。

## 7. 验收标准

九个引导节点正确触发且可跳过、保存与重置；结算信息可靠；默认固定 Seed 可复现；草稿试玩不污染正式配置和存档；桌面和移动端可完成核心流程；旧存档和旧配置兼容；全部自动测试与生产构建通过；README 与当前状态文档同步实际实现。
