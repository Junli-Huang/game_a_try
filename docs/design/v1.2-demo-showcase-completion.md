# 《雾下余粮》V1.2：Demo 可展示版本

> 文档状态：开发基线  
> 依赖版本：V1.1.1 交互与战斗表现优化  
> 目标：让新玩家在 5～10 分钟内理解目标、完成一次外出，并感受到饥饿、怪物肉和疯狂之间的矛盾。

## 1. 范围

本版本不扩张装备、制作、Boss、多地图、联机或后端系统，只补齐展示所需的目标、氛围、探索变化、反馈与音频接口。

## 2. Demo 目标与结算

默认目标是在安全食物耗尽前完成 3 次成功撤离，并累计带回 12 块怪物肉。成功撤离和肉量必须同时满足。

失败条件为：外出失败达到 3 次，或庇护所安全食物与怪物肉同时耗尽。目标、肉量和失败上限全部配置化。

新存档第一次进入庇护所显示目标介绍；庇护所显示完整进度，户外 HUD 显示简化进度。胜利与失败在返回庇护所时结算，不强制中断当前外出。

存档新增：成功撤离、失败次数、累计带回肉量、击杀、切割、最高疯狂、介绍与结算状态、永久事件记录。旧存档通过默认值合并升级。

## 3. 疯狂表现

疯狂阶段继续决定攻击倍率，同时驱动表现层：

- 清醒：稳定画面。
- 低语：轻微紫红边缘和低语文字。
- 狂热：边缘增强、疯狂数字脉动、敌人对比度提高。
- 失控边缘：缓慢边缘呼吸和不改变点击区域的轻微 HUD 偏移。

跨阶段显示独立文案；食用怪物肉按当前阶段选择不同描述；同一低语不连续重复。所有动画支持“减少动态效果”，表现层关闭后不影响规则。

## 4. 地图事件

玩家首次进入普通格时按配置概率判定。单次外出默认最多 3 次，事件之间至少间隔 5 步；战斗、切割、撤离和事件弹窗期间不重复触发。

第一批事件：废弃背包、未熄灭的火堆、残缺笔记、腐烂储藏箱、远处脚步、陌生尸体、雾中呼唤、熟悉味道。事件支持权重、疯狂条件、单次/永久限制、选择、随机结果和立即效果。

## 5. 音频

`AudioService` 提供 BGM、音效、静音和三级音量接口。当前使用 Web Audio 合成短提示音，不依赖外部资源；首次用户操作后解锁音频，浏览器拒绝播放或缺少音频能力时静默失败，不阻塞游戏。

## 6. UI

- 主菜单：开始、继续、配置、关于、重置存档。
- 庇护所：目标进度、最近结果、库存、装备、种植和外出入口。
- 户外：不遮挡地图的 HUD、目标简报、本次背包、上下文交互按钮。
- 战斗：双方生命/速度、先手与行动方、日志与高亮按钮。
- 全局：叙事弹窗、事件选择、结果面板、低语和疯狂阶段提示。

移动端使用动态视口单位与安全区，地图保持完整正方形，触摸按钮不小于约 44px。

## 7. 配置增量

新增 `demoGoal`、`madnessPresentation`、`mapEvents`、`events`、`audio`，并扩充 `ui`。全部字段进入默认配置、本地保存、JSON 导入导出、恢复默认和校验流程。

## 8. 验收

新存档可完成：查看目标 → 外出 → 探索/事件 → 感知提示 → 战斗 → 切割 → 撤离 → 目标进度。刷新后进度不丢失；关闭音频和疯狂视觉后规则正常；桌面和手机均能完整看到地图与关键操作。

## 9. V1.2.1 操作与资源规则补充

- 战斗行动菜单为“攻击 / 防御 / 道具 / 逃跑”。异变肉块属于道具，必须先进入“道具”二级列表后选择。
- 战斗菜单和道具列表支持键盘操作：方向键移动焦点，回车确认；鼠标和触屏继续可用。
- 户外探索界面提供“使用道具”入口，显示异变肉数量与效果。
- 饥饿值只在玩家于户外地图移动时扣除。战斗、切割、等待、撤离读条和使用道具均不产生饥饿消耗。
