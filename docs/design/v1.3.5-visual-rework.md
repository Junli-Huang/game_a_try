# V1.3.5 Visual Rework

## 目标

户外探索仍严格使用格子移动与离散回合，但视觉上弱化“棋盘格”感，让地面、敌人感知与战争迷雾更接近阴冷、潮湿的荒野。

本版本只改变渲染，不修改移动、寻路、视野判定、敌人状态机、地图配置或存档格式。

## 视觉规则

### 地表与格线

- 地面使用低对比色块和确定性微纹理，不把生成式参考图直接平铺到每个格子。
- 格线使用低透明度虚线，只作为移动尺度提示。
- 两个相邻障碍之间不绘制共享格线，让连续障碍形成整体轮廓。
- 障碍与可行走地面之间仍保留淡线，以保证碰撞边界可读。

### 敌人方向视野

- 视野由 Canvas 绘制连续弧形扇面，不再逐格填充独立色块。
- 扇面的角度、半径和朝向直接来自现有方向视野配置。
- 障碍遮挡仍使用现有 `getVisionCells()` 结果作为裁剪区域，因此视觉不会越过实际可见范围。
- Idle、Alert、Chase / AttackIntent 和 Cooldown 使用同一图形、不同强度色调。
- 敌人本体边缘显示随朝向旋转的小型三角指针；记忆状态、尸体和巢穴不显示实时方向。

### 战争迷雾

- 未探索区域仍是不可穿透的实体暗层，不透露地图内容。
- 在当前可见或已探索区域与未探索区域的交界处，叠加由固定坐标生成的不规则柔化雾团。
- 雾团不会随渲染随机跳动；相同地图位置每次绘制保持一致。
- 已探索记忆仍使用现有亮度配置，不更新离开视野后的敌人实时状态。

## Asset 使用原则

视觉资源包中的整体效果图、地表和雾团原型用于色调与质感参考。精确影响规则的格线、扇形视野、方向标记和战争迷雾全部由 Canvas 程序化绘制，以适配任意地图尺寸、GitHub Pages 子路径及移动端缩放。

当前版本没有把带绘制背景的原型裁片作为正式透明素材引入运行时，避免出现接缝、背景边和错误遮挡。

## 验收

- 移动、接敌、障碍遮挡和视野检测与 V1.3.4 一致。
- 地图仍能读出移动尺度，但不再呈现高对比实体格框。
- 相邻障碍内部没有格线。
- 当前可见敌人的视野是连续扇面，且朝向指针紧贴本体旋转。
- 未探索区保持完全遮挡，边界具有不规则柔化雾沿。
- 固定视觉扰动可重复，连续重绘不闪烁。
- 桌面与手机端画布保持正方形和清晰缩放。
