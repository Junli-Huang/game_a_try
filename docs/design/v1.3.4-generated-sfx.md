# V1.3.4 Generated SFX

## 1. 目标与风格

本版本将单音正弦提示升级为可提交、可重复生成的真实 WAV 资源。声音保持克制、阴冷、潮湿、低保真和轻微有机感，不包含 BGM，也不依赖在线服务或外部素材。

生成脚本组合三角/正弦音色、固定 Seed 噪声、音高滑动、短包络、低通与轻度软削波。所有文件为 44100 Hz、16-bit PCM、Mono，峰值归一化到低于 -1 dBFS。

## 2. 资源与生成

资源存放在 `public/assets/audio/sfx/`，脚本为 `scripts/generate-sfx.mjs`：

```bash
npm run generate:sfx
```

固定 `SFX_SEED = 1334`。重复运行生成相同字节；生产构建直接复制已提交的 WAV，不在 Pages 构建现场合成。

文件清单：

```text
ui-click-01.wav
player-step-01.wav / player-step-02.wav
player-attack-01.wav / player-hurt-01.wav / player-defend-01.wav
enemy-alert-01.wav / enemy-chase-01.wav / enemy-attack-intent-01.wav
battle-enter-01.wav / battle-victory-01.wav / battle-defeat-01.wav / battle-escape-01.wav
corpse-harvest-01.wav / item-obtain-01.wav / monster-meat-eat-01.wav
madness-rise-01.wav / resistance-depleted-01.wav
relic-purify-01.wav / resistance-restore-01.wav
extraction-progress-01.wav / extraction-success-01.wav
```

## 3. 事件映射

| ID | 游戏事件 |
| --- | --- |
| `click` | 模态框与 UI 确认 |
| `move` | 玩家成功移动一格 |
| `attack` / `hurt` / `defend` | 玩家攻击、受伤、防御承伤 |
| `alert` / `chase` / `intent` | 敌人首次状态通知 |
| `battle` | 战斗转场 |
| `victory` / `failure` / `escape` | 战斗结果 |
| `harvest` / `item` | 切割完成、获得物品 |
| `eat` / `madness` | 食用异变肉、有意义的疯狂增长 |
| `resistance_depleted` | 单次外出抗性首次耗尽 |
| `purify` / `resistance_restore` | 圣遗物净化肉、恢复抗性 |
| `extract` / `extract_success` | 开始撤离、撤离成功 |

敌人转向不播放声音。状态音由状态变化通知触发，不由渲染触发；撤离开始音只在调用撤离动作时触发，进度回合不会重复播放；快照恢复和配置迁移不会补播。

## 4. AudioService

`AudioService` 提供 `preload()`、`unlock()`、`playSfx()`、静音与音量设置、`dispose()`，并保留空的 BGM 接口。

- `masterVolume × sfxVolume × 单音效音量` 共同生效。
- 脚步在两个文件间轮换，并支持小幅音高、音量变化。
- 每个事件有最低播放间隔，抑制连续状态刷新和节点堆积。
- 播放节点结束后从活动集合移除；服务重建时停止活动节点并关闭旧上下文。
- 文件加载失败不阻塞游戏；开启 `fallbackSynthEnabled` 时使用短促三角波回退。
- `useGeneratedAssets = false` 时不加载 WAV，但仍可使用配置允许的合成回退。

资源地址按 `${import.meta.env.BASE_URL}assets/audio/sfx/${filename}` 组成，因此开发环境和 `/game_a_try/` 子路径均可用。

## 5. 浏览器限制与验收

浏览器自动播放策略要求用户交互后恢复 `AudioContext`。页面同时监听首次指针和键盘交互；页面加载、存档恢复和配置迁移不会主动播放。

自动验收命令：

```bash
npm run generate:sfx
npm test
npm run build
```

桌面、移动端实际响度、扬声器差异与长时间连续战斗仍需要人工试听。本版本没有 BGM。
