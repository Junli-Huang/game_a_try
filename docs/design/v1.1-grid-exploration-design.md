# 《雾下余粮》V1.1：格子探索与回合制战斗设计

> 文档状态：V1.1 开发基线  
> 变更范围：户外探索、战争迷雾、敌人地图 AI、接敌、战斗、切割与撤离  
> 保留系统：主菜单、庇护所、食物、疯狂、装备、种植、库存、本地存档与配置服务

## 1. 调整结论

户外部分由“即时移动、即时战斗”调整为两个明确层次：

> **20×20 格子地图探索 + 接敌后独立回合制战斗。**

调整后的重点从动作操作转向：

- 路线规划。
- 视野管理。
- 对敌人位置的记忆。
- 判断旧情报是否仍可信。
- 主动接敌或绕开敌人。
- 根据生命、食物和疯狂决定是否继续深入。
- 在切割与撤离期间承担时间风险。

这个方向继续服务于原核心矛盾：安全食物有限，怪物肉容易获得但会增加疯狂，疯狂又会提高玩家战斗能力。

## 2. 户外流程

1. 玩家从庇护所开始外出。
2. 进入一张 20×20 户外地图。
3. 玩家出生在指定格子。
4. 玩家逐格移动并探索战争迷雾。
5. 当前视野内可看见明雷敌人。
6. 玩家或敌人主动接触后切换到独立战斗画面。
7. 战斗胜利后回到原地图，敌人原格生成尸体。
8. 玩家在尸体格执行切割，获得怪物肉。
9. 玩家继续探索或前往撤离点。
10. 玩家在撤离点消耗若干地图回合完成撤离。
11. 本次获取的资源转入庇护所。

户外模块明确划分为：

```text
SHELTER
OUTDOOR_EXPLORATION
BATTLE
OUTDOOR_RESULT
EXTRACTION_RESULT
```

## 3. 20×20 格子地图

V1.1 默认尺寸：

```text
width = 20
height = 20
x = 0～19
y = 0～19
```

每个格子至少包含：

- 坐标。
- 地形类型。
- 是否可通行。
- 可见状态。
- 当前真实内容。
- 玩家最后看见的内容快照。
- 是否为出生点。
- 是否为撤离点。
- 是否为种植点。

推荐模型：

```ts
interface TileState {
  x: number;
  y: number;
  terrainId: string;
  walkable: boolean;
  visibility: "unexplored" | "explored" | "visible";
  currentContent: TileContent;
  rememberedContent: TileContent | null;
}

interface TileContent {
  enemyId?: string;
  corpseId?: string;
  interactableId?: string;
  cropState?: string;
}
```

### 3.1 V1.1 地形

第一版只要求：

- 普通地面：可通行。
- 障碍：不可通行。
- 玩家出生点。
- 撤离点。
- 户外种植点结构预留。

森林、水域、沼泽、废墟、污染区、高地和事件格延后。

## 4. 玩家移动与地图回合

玩家只支持四方向移动：上、下、左、右；每次只能进入相邻可通行格。

操作方式：

- WASD。
- 方向键。
- 点击相邻格。
- 移动端方向按钮继续保留。

每次移动按以下顺序结算：

1. 检查目标格是否越界或不可通行。
2. 判断目标格是否存在敌人；存在则先进入战斗，不直接覆盖。
3. 更新玩家坐标。
4. 更新视野和战争迷雾。
5. 推进敌人地图 AI。
6. 判断敌人是否主动接敌。
7. 结算行动消耗、饥饿和持续效果。
8. 刷新地图显示。

玩家不操作时，户外世界不自动推进。

```text
玩家移动一次 = 1 个地图回合
```

切割、等待和撤离同样消耗地图回合，并允许其他敌人在每个回合行动。

## 5. 三态战争迷雾

战争迷雾采用类似《星际争霸》的三态机制。

### 5.1 UNEXPLORED：未探索

- 完全遮挡。
- 不显示地形、敌人、尸体、种植和可交互物。
- 不保存可展示快照。

### 5.2 EXPLORED：已探索但当前不可见

- 显示最后一次观察到的地形和环境。
- 颜色变暗、降低饱和度。
- 不接收视野外的后续变化。
- 敌人、尸体等动态对象显示为记忆标记，而非实时信息。

核心规则：

> 玩家记得自己曾经看到的内容，但不知道离开视野后发生了什么。

### 5.3 VISIBLE：当前可见

- 显示真实地形。
- 显示敌人真实位置。
- 显示尸体、种植和最新变化。
- 每次进入可见状态时，用真实内容刷新记忆快照。

格子离开视野时保留最后一次快照；格子在视野外发生变化时，快照不更新。

### 5.4 视野

V1.1 默认：

- 基础视野半径 3 格。
- 使用曼哈顿距离或简单方形范围。
- 地形暂不遮挡视野。
- 允许看到斜向格，但不允许斜向移动。

后续可加入墙体遮挡、森林和夜晚修正、装备加成及高疯狂导致的错误记忆。

## 6. 明雷敌人与地图 AI

敌人直接存在于地图格上。只有当前可见时显示真实位置；已探索区域可以显示半透明旧位置，明确标记为记忆而非实时情报。

敌人的 AI 使用真实地图状态，不受玩家战争迷雾影响。玩家视野外的敌人仍可游荡、追踪和返回出生点，但玩家不会获知这些变化。

### 6.1 统一地图状态

- `Idle`：待机。
- `Wander`：游荡。
- `Chase`：追踪玩家。
- `Return`：返回出生区域、丧失战斗意志。
- `Cooldown`：脱战后的短暂冷却。
- `Dead`：已从 AI 列表移除，地图上留下尸体。

三种敌人继续使用统一参数与状态机，不拆成三套类型代码。

### 6.2 静止型

- 永远停留在出生格。
- 不主动接近玩家。
- 玩家尝试进入其所在格时触发战斗。

### 6.3 游荡者

- 在出生点附近随机移动。
- 每回合按配置概率决定是否行动。
- 一次最多移动配置的格数，V1.1 默认 1 格。
- 感知玩家后主动靠近。
- 超出活动范围后返回。

### 6.4 追踪者

- 初始可以静止，也可以配置为游荡。
- 玩家进入感知范围后开始追踪。
- 每回合向玩家靠近。
- 超过追踪限制或最大离家距离后返回出生点。

## 7. 接敌规则

### 7.1 玩家主动接敌

玩家尝试进入敌人格时：

1. 玩家暂不进入目标格。
2. 保存玩家、敌人和地图状态。
3. 切换到战斗画面。
4. 玩家获得先手。

### 7.2 敌人主动接敌

敌人尝试进入玩家格时：

1. 敌人暂不覆盖玩家格。
2. 保存接敌来源与地图状态。
3. 切换到战斗画面。
4. 敌人获得先手。

V1.1 不在地图上实现远程攻击。主动发起接触的一方获得先手，规则可配置。

## 8. 独立回合制战斗

战斗界面显示：

- 玩家与当前敌人。
- 双方生命与状态。
- 行动菜单。
- 战斗日志。
- 当前疯狂阶段与攻击倍率。

每个战斗回合：

1. 玩家选择行动。
2. 结算玩家行动。
3. 判断敌人是否死亡。
4. 敌人选择并结算行动。
5. 判断玩家是否死亡。
6. 结算饥饿消耗。
7. 进入下一回合。

V1.1 玩家行动：

- 攻击。
- 防御。
- 使用怪物肉。
- 逃跑。

### 8.1 攻击

攻击伤害由玩家基础攻击、默认装备、疯狂阶段和防御共同决定。玩法层不得写死具体数值。

### 8.2 防御

防御只持续到本次敌人行动，按配置比例降低受到的伤害。

### 8.3 使用食物

是否允许在战斗中使用食物由配置决定。食用怪物肉恢复饥饿并增加疯狂。

### 8.4 逃跑

V1.1 使用可配置的基础成功率。

逃跑成功：

- 返回接敌前的玩家格。
- 敌人回到接敌前位置。
- 双方生命变化保留。
- 敌人进入短暂脱战冷却，避免立即重复接敌。

逃跑失败：敌人立即执行一次行动，是否立即攻击由配置控制。

战斗模块只返回结果，不直接修改地图显示：

```ts
interface BattleResult {
  result: "victory" | "defeat" | "escaped";
  playerState: PlayerState;
  enemyState: EnemyBattleState;
}
```

## 9. 战斗结果回写地图

### 9.1 胜利

- 敌人从地图 AI 列表移除。
- 敌人原格生成尸体。
- 玩家进入敌人原格。
- 该格可以同时包含玩家和尸体。
- 保留战争迷雾与地图回合状态。

### 9.2 失败

- 本次外出结束。
- 丢失本次获得的资源。
- 默认装备保留。
- 疯狂值按配置决定是否保留。
- 本次户外实例销毁。

### 9.3 逃跑

- 返回接敌前状态。
- 双方当前生命保留。
- 敌人进入配置的脱战冷却回合。

## 10. 尸体与切割

敌人死亡后，尸体位于敌人原格。尸体只有在当前视野中显示真实状态；已探索区域显示最后一次观察快照。

V1.1 切割改为离散回合行为：

- 玩家位于尸体格时可以切割。
- 切割一次性开始，消耗配置的若干地图回合。
- 每个切割回合都会推进其他敌人行动和饥饿消耗。
- 敌人接触玩家时切割中断并进入战斗。
- 完成后获得配置数量的怪物肉。
- 尸体变为已切割状态或移除。

默认消耗：静止型 2 回合，游荡者 3 回合，追踪者 4 回合。

## 11. 行动与饥饿消耗

实时饥饿消耗被离散行动消耗替代。

配置字段：

```text
hungerCostPerMove
hungerCostPerBattleRound
hungerCostPerHarvestRound
hungerCostPerWait
```

移动、战斗、切割、撤离和等待都可以产生消耗。玩家不操作时不扣除饥饿。

## 12. 撤离

撤离点是特殊地图格。玩家进入撤离格后执行撤离操作，连续消耗配置的地图回合。

- 每个撤离回合都会推进敌人 AI。
- 敌人接触玩家时撤离中断并进入战斗。
- 完成后将本次资源带回庇护所。
- V1.1 默认撤离耗时 3 个地图回合。

## 13. 户外种植

V1.1 保留户外种植格类型和配置接口，但不实现完整跨局种植。

庇护所种植继续按原设计运行。户外种植的播种、污染、破坏、跨局生长和收获延后。

## 14. 配置系统调整

配置页面继续定义游戏规则，不提供 GM 操作。保存后在下一次开始游戏时生效。

### 14.1 户外地图

- 地图宽度和高度。
- 是否允许斜向移动。
- 玩家出生点。
- 撤离点。
- 撤离所需回合。
- 障碍格。
- 怪物出生点和怪物类型。
- 户外种植点结构。

### 14.2 视野与战争迷雾

- 是否启用战争迷雾。
- 玩家视野半径。
- 视野形状。
- 地形是否遮挡。
- 已探索区域亮度。
- 是否显示敌人记忆标记。
- 是否显示尸体记忆标记。

### 14.3 敌人地图行为

- 每回合行动概率。
- 每回合最大移动格数。
- 游荡半径。
- 感知距离。
- 追踪和离家距离。
- 返回规则。
- 是否主动接敌。
- 主动接敌是否先手。
- 脱战冷却回合。

### 14.4 回合制战斗

- 玩家可用行动。
- 主动接敌先手规则。
- 基础逃跑概率。
- 逃跑失败是否立即攻击。
- 战斗回合饥饿消耗。
- 防御减伤比例。
- 是否允许战斗中使用食物。
- 胜利后玩家位置规则。

## 15. 配置地图预览

V1.1 配置页至少提供简单 20×20 网格预览，属于配置编辑器而非 GM 工具。

应支持：

- 查看地图尺寸、出生点、撤离点、障碍和怪物出生点。
- 点击设置出生点。
- 点击设置撤离点。
- 点击切换障碍。
- 点击放置所选怪物。
- 点击删除实体。

不要求完整关卡编辑器、拖拽、多选、刷子尺寸或运行时实体调试。

## 16. V1.1 必须实现

- 20×20 格子地图。
- 玩家四方向移动和点击相邻格。
- 不可通行障碍。
- 三状态战争迷雾。
- 玩家视野范围。
- 离开视野后保留最后观察快照。
- 明雷敌人和旧位置记忆标记。
- 三种参数化敌人模板。
- 敌人按地图回合移动、追踪和返回。
- 玩家或敌人接触后切换独立回合制战斗。
- 攻击、防御、使用食物和逃跑。
- 主动接敌方先手。
- 战斗胜利、失败和逃跑。
- 胜利后生成尸体。
- 尸体切割并获得怪物肉。
- 撤离点与离散撤离回合。
- 返回庇护所和原有资源结算。
- 相关配置字段和地图网格预览。

## 17. V1.1 暂缓

- 斜向移动。
- 地形移动消耗差异。
- 复杂视线遮挡。
- 多敌人同时战斗和阵型。
- 技能与复杂异常状态。
- 随机地图和动态刷怪。
- 跨局户外地图持久化。
- 户外种植完整流程。
- 敌人之间互相战斗。
- 伏击、背刺和远程地图攻击。
- 多个撤离点。
- 昼夜和天气。
- 高疯狂导致的错误地图记忆。

## 18. 验收标准

1. 玩家可以从庇护所进入 20×20 地图并逐格移动。
2. 玩家不操作时地图状态、敌人和饥饿不变化。
3. 未探索、已探索、当前可见三种状态表现明确。
4. 已探索区域只显示最后一次观察内容，不泄漏实时变化。
5. 当前可见敌人以明雷显示，接触后进入独立战斗界面。
6. 主动接敌的一方先手。
7. 战斗胜利后正确回写尸体、玩家位置和敌人移除状态。
8. 切割和撤离会推进地图回合，并可能被敌人接触打断。
9. 怪物肉、疯狂、死亡损失和庇护所库存继续正确结算。
10. 配置页能调整地图、视野、敌人地图行为和战斗规则，并在下一局生效。
