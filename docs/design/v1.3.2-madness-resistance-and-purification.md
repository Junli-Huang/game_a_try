# 《雾下余粮》V1.3.2 疯狂抗性、圣遗物与怪物肉净化

## 1. 版本目标

V1.3.2 在现有 V1.3.1 核心循环上增加一层可持续的污染管理：玩家在户外会持续承受环境污染，先消耗疯狂抗性，回到庇护所后再决定将有限的圣遗物净化值用于恢复抗性，还是净化带回的异变肉。

本版本不增加新地图、怪物或独立玩法系统。

## 2. 核心循环增量

```text
进入户外并保留当前生命、疯狂和疯狂抗性
→ 环境每 5 秒施加 0.1 点污染
→ 污染先消耗疯狂抗性，溢出部分进入玩家疯狂
→ 战斗与切割获得带独立污染值的异变肉
→ 撤离时保留生命、疯狂、抗性和每块肉的污染状态
→ 在庇护所用圣遗物恢复抗性或净化异变肉
→ 食用异变肉时仅吸收该肉当前剩余污染
```

## 3. 玩家疯狂抗性

- 玩家配置新增 `maxMadnessResistance` 与 `initialMadnessResistance`，默认均为 `10`。
- 新游戏的当前疯狂抗性为 `10`，当前值和最大值进入存档。
- 进入户外、成功撤离或失败返回均不会自动恢复抗性。
- 疯狂抗性只抵消户外环境污染，不降低已经存在的疯狂，也不抵消食用异变肉的污染。
- 旧存档缺少字段时，以当前配置默认值补齐。

环境污染结算：

```text
抵消污染 = min(当前疯狂抗性, 本次污染)
剩余污染 = 本次污染 - 抵消污染
当前疯狂抗性 -= 抵消污染
当前疯狂 += 剩余污染
```

数值保留四位小数进行结算，界面最多显示两位小数。

## 4. 户外环境污染

每张地图使用独立配置：

```js
environmentMadness: {
  enabled: true,
  amount: 0.1,
  intervalSeconds: 5
}
```

- 只在运行中的 `OUTDOOR_EXPLORATION` 模式计时。
- 引导暂停、战斗、战斗转场、地图事件等非户外探索状态不计时。
- 一次更新跨过多个间隔时会补齐所有应结算次数。
- 外出快照保存当前间隔内的毫秒进度和“抗性耗尽”提示状态；刷新仍处于未结算外出时，会从原进度继续计时，不把页面关闭或不可见期间当作有效探索时间。
- 成功撤离或失败后随外出快照一起清除计时状态。

## 5. 怪物肉污染实例

异变肉不再只保存总数量。每块肉保存：

```js
{
  id,
  currentMadness,
  maxMadness
}
```

- `monsterMeat.maxMadness` 默认是 `12`。
- 切割或地图事件生成新肉时，`currentMadness = maxMadness`。
- 户外背包和庇护所库存都保存完整肉块对象。
- 撤离保留每块肉的污染；失败仍按原规则丢失本次外出获得的肉。
- 食用时优先选择当前污染最低的一块，方便玩家实际消费已净化成果。
- 食肉增加的疯狂等于该肉的 `currentMadness`，不会被玩家抗性抵消；地图与战斗使用同一逻辑。
- 旧存档的整数肉量迁移为等量、满污染的肉块对象，庇护所库存和户外快照均不会丢失数量。

## 6. 庇护所圣遗物

默认配置：

```js
relic: {
  name: '静默圣遗物',
  enabled: true,
  maxPurification: 100,
  initialPurification: 100,
  resistanceRestoreCostMultiplier: 1,
  meatPurificationCostMultiplier: 1,
  protectsShelter: true
}
```

仓库沿用既有配置分类名称 `relic`，对应策划中的 `shelterRelic`。圣遗物在庇护所中可见，并显示当前/最大净化值、玩家抗性、庇护屏障状态和预计消耗。默认倍率均为 1，当前版本提供两个可配置倍率的转换操作：

1. 恢复疯狂抗性：可恢复量为 `min(抗性缺口, 当前净化值 / 恢复倍率)`，实际消耗为恢复量乘以倍率。
2. 净化一块肉：玩家从污染肉列表中明确选择对象并确认；可净化量为 `min(肉当前污染, 当前净化值 / 净化倍率)`，允许部分净化且不会改变最大污染。

倍率为 0 时对应操作不消耗净化值。抗性已满、没有受污染的肉或可净化量为零时按钮禁用且不发生消耗；圣遗物当前净化值不会低于零。

## 7. 数值转移原则

V1.3.2 明确记录以下转移或转换：

- 环境污染先与玩家疯狂抗性结算，未被抵消的部分进入玩家疯狂。
- 食用异变肉时，肉块的当前污染成为玩家新增疯狂。
- 圣遗物恢复抗性或净化肉时，按实际处理量乘以配置倍率消耗净化值；默认倍率为 1:1。
- 战斗中玩家和怪物实际损失的 HP 由当前场景按 1:1 吸收并记录为 `sceneMadness`，过量伤害只按实际损失 HP 计算。

当前户外场景仍随每次外出实例化，`sceneMadness` 保存于本次外出快照并显示在探索结算中，外出结束后不永久保存。场景永久化与跨场景污染总账留待后续版本。

## 8. 存档与配置迁移

- 玩家存档保存生命、疯狂、当前/最大疯狂抗性、圣遗物、庇护所肉块实例和户外快照。
- 户外快照保存玩家抗性、肉块实例、环境污染计时进度和场景疯狂。
- 旧配置补齐玩家抗性、地图环境污染、怪物肉和圣遗物配置。
- 旧玩家存档补齐抗性和圣遗物。
- 旧整数肉量按数量迁移为满污染肉块，庇护所与户外数据分别迁移。
- 迁移不会改变生命、疯狂、外出统计、地图实体或其他既有存档数据。

## 9. 明确不做

- 主动污染怪物肉。
- 圣遗物充能、升级或多个圣遗物。
- 庇护所屏障失效。
- 通用 Buff/Debuff 框架。
- 完整通用物品系统。
- 新怪物、地图或大型玩法。
- 场景永久化与跨场景污染总账。

## 10. 验收标准

1. 新游戏疯狂抗性默认 10/10。
2. 户外默认每 5 秒结算 0.1 点污染。
3. 污染先扣抗性，抗性不足时仅将剩余值加入疯狂。
4. 食肉污染不被抗性抵消。
5. 抗性不会因场景切换、撤离或失败返回自动恢复。
6. 圣遗物可见并显示当前/最大净化值。
7. 圣遗物可以足量或部分恢复抗性、净化异变肉，数值不低于零。
8. 每块肉保存当前/最大疯狂，食用只增加当前污染。
9. 战斗和地图食肉结果一致。
10. 生命、疯狂、抗性、圣遗物、肉块污染和户外污染计时正确保存。
11. 刷新后继续污染计时，旧存档肉量不丢失。
12. 战斗实际 HP 损失按 1:1 计入当前场景疯狂。
13. 自动测试与生产构建通过。
14. `current-game-status.md` 与实际代码一致。
