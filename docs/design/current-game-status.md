# 《雾下余粮》当前游戏状态

> 本文档是当前实现状态的单一事实来源。
> 当本文档与 V1、V1.1、V1.1.1、V1.2 设计文档存在冲突时，以本文档和实际代码行为为准。
>
> 历史设计文档用于记录设计演进，不代表所有内容均已实现。

- 当前版本：V1.2.1 Demo
- 状态依据：`main` 分支实际代码、自动测试与生产构建
- 在线版本：[GitHub Pages](https://junli-huang.github.io/game_a_try/) / [chatgpt.site](https://tiny-signal-game.junli-h.chatgpt.site)
- 更新规则：功能验收后，必须同步更新本文档；不能只更新历史设计文档。

## 1. 当前核心循环

```text
主菜单
→ 庇护所查看目标、库存、装备和种植
→ 进入 20×20 户外地图
→ 探索迷雾、处理随机事件、观察敌人意图
→ 接敌并进入独立回合制战斗
→ 击杀后切割尸体获得异变肉
→ 选择继续深入、食用异变肉或前往撤离点
→ 成功撤离后将本次异变肉带回庇护所
→ 推进种植、Demo 目标与胜负结算
```

默认 Demo 胜利条件：成功撤离 3 次，并累计带回至少 12 块异变肉。

默认失败条件：外出失败达到 3 次，或庇护所的储备粮与异变肉同时耗尽。

## 2. Implemented

### 2.1 主菜单、存档与结算

- 开始游戏、继续游戏、配置、关于、重置存档。
- 存档使用浏览器 `localStorage`，保存库存、疯狂、种植、外出次数、目标进度和统计数据。
- 旧存档通过默认字段合并升级。
- 首次目标说明、胜利/失败结算、继续游戏与重新开始。

### 2.2 庇护所

- 查看疯狂阶段、固定默认装备、食物库存、最近一次外出结果和 Demo 目标。
- 食用储备粮或异变肉；异变肉会增加疯狂。
- 一个灰麦种植格：消耗储备粮播种，按外出次数生长，成熟后产出储备粮。
- 从庇护所进入唯一的户外区域。

### 2.3 户外地图与战争迷雾

- 一张 20×20 四方向格子地图，包含出生点、障碍、怪物刷新点和固定撤离点。
- 方向键或 WASD 移动，也可以点击相邻格；手机提供屏幕方向键。
- 三态战争迷雾：未探索、已探索记忆、当前可见。
- 离开视野后保留最后观察到的敌人与尸体记忆，不继续刷新记忆画面。
- 地图采用离散回合：玩家移动、等待、切割或撤离读条会推进敌人行动。

### 2.4 敌人与危险反馈

- 三套数据预设：静默菌兽、游荡者、寻迹者。
- 共用统一参数化状态机：`Idle / Wander / Alert / Chase / AttackIntent / Return / Cooldown / Dead`。
- 支持感知、警觉延迟、追踪、攻击意图、离家限制、返回和脱战冷却。
- 敌人发现、追踪和即将接敌时提供文字、颜色与危险提示。

### 2.5 战斗

- 接敌冻结地图，播放敌人特写转场，再进入独立战斗画面。
- 显示双方生命、攻击、速度、疯狂、回合与战斗日志。
- 速度较高的一方在战斗开始时先行动。
- 玩家行动：攻击、防御、道具、逃跑。
- “道具”进入二级菜单；当前仅包含本次外出获得的异变肉。
- 战斗键盘输入接入游戏运行时：方向键或 WASD 移动选项，回车或空格确认。
- 当前选项显示金色高亮；鼠标和触屏仍可直接操作。
- 胜利留下尸体；失败返回庇护所；逃跑使敌人暂时进入冷却状态。

### 2.6 饥饿、疯狂与道具

- 新局每次进入户外时，生命与饥饿按玩家配置初始化；默认饥饿为 100/100。
- 只有在户外地图实际移动才扣除饥饿，默认每格消耗 1。
- 战斗、等待、切割、撤离读条、事件推进和使用道具不扣除饥饿。
- 饥饿归零后，后续移动会受到饥饿伤害。
- 户外“使用道具”入口和战斗“道具”菜单均可食用异变肉。
- 异变肉恢复饥饿并增加疯狂；疯狂阶段提高攻击倍率。
- 疯狂阶段：清醒、低语、狂热、失控边缘；包含阶段文案、低语、边缘效果和减少动态效果选项。

### 2.7 尸体、切割与撤离

- 怪物死亡后在地图对应格留下尸体。
- 站在尸体格可点击高亮按钮或按 E 切割。
- 切割按怪物配置推进多个地图回合，并受背包容量、装备和切割倍率影响。
- 当前尸体资源只有异变肉。
- 站在撤离点可点击高亮按钮或按 E，完成配置回合数后撤离。
- 成功撤离保留本次肉；失败默认丢失本次肉，固定装备保留，疯狂按配置保留。

### 2.8 随机事件

- 8 个数据驱动事件：废弃背包、未熄灭的火堆、残缺笔记、腐烂储藏箱、远处脚步、陌生尸体、雾中呼唤、熟悉味道。
- 支持启用开关、权重、疯狂条件、事件间隔、单次外出上限、选择和随机结果。
- 事件可以恢复生命/饥饿、增加疯狂、给予食物或推进地图回合。
- 负数饥饿事件效果不生效，饥饿扣除仍只来自地图移动。

### 2.9 配置系统

- 分类：全局、玩家、怪物、食物、疯狂、装备、地图、战斗、UI、Demo 目标、疯狂表现、地图事件、音频、种植、庇护所。
- 支持表单编辑、本地保存、应用、恢复默认、基础校验、JSON 导入与导出。
- 配置在下一次开始游戏/进入流程时加载，不支持运行中完整热更新。
- 旧战斗行动 `eat` 自动迁移为 `item`。

### 2.10 展示与部署

- 桌面与移动端响应式布局，移动端地图保持完整正方形并考虑安全区。
- Web Audio 合成操作、警觉、战斗、切割、撤离与结果提示音。
- 自动测试覆盖配置、迷雾记忆、战斗、敌人意图、道具、饥饿、目标、疯狂和事件。
- GitHub Pages 与 chatgpt.site 两套公开部署。

## 3. Partially Implemented

### 3.1 庇护所恢复与食物

- 庇护所有食用按钮和库存扣除逻辑。
- 异变肉在庇护所会增加疯狂。
- 生命与饥饿没有作为跨外出状态持久化；每次外出按玩家配置重新初始化为满生命和默认饥饿。
- 因此“在庇护所通过食物恢复下一次外出的饥饿”尚未形成完整玩法，储备粮目前主要用于失败条件和种植投入。

### 3.2 装备

- 装备数据、默认装备、攻击、防御和切割产量加成已经接入。
- 当前只能使用配置中标记为默认装备的固定组合。
- 没有游戏内换装、装备库存、掉落、耐久、词条或成长。

### 3.3 道具与背包

- 有背包容量和道具入口，但运行时背包当前只存储异变肉。
- 储备粮配置声明可在户外食用，但目前不会被带入户外背包。
- 没有通用道具实例、堆叠管理、丢弃、排序或多类型搜刮。

### 3.4 音频

- 短提示音、静音和音量配置已实现。
- `AudioService` 保留 BGM 接口，但当前没有实际背景音乐资源与循环播放。

### 3.5 配置编辑能力

- 能修改现有配置和嵌套数据，并提供地图格预览。
- 尚未形成完整的可视化地图编辑器。
- 怪物/装备/事件结构允许扩展，但新增、复制、删除等专用内容管理体验仍较基础。

## 4. Reserved

- `enableOutdoorFarming`、作物 `allowOutdoor`：为户外种植预留，默认关闭。
- 怪物 `carriedLoot`：为怪物携带物掉落表预留，当前为空。
- 多件装备配置：为后续装备选择预留，当前只读取默认装备。
- `AudioService.playBgm/stopBgm`：为背景音乐预留。
- 事件的条件、权重和永久记录字段：可用于继续增加事件内容。
- 配置版本、云端配置、多套配置方案：架构可扩展，但当前只使用本地存储。
- 胜利后的经验、特殊资源和成长展示位置：只保留表现方向，没有数值系统。

Reserved 只表示数据结构或接口已经留位，不代表对应玩法可用。

## 5. Not Implemented

- 服务端、账号、云存档、排行榜、留言或实时联机。
- 多张户外地图、随机地图生成、地图选择和完整关卡编辑器。
- 户外种植、怪物破坏作物和污染作物生态。
- 装备掉落、换装、耐久、随机词条、制作、交易和经济系统。
- 枪械、弹药、远程战斗、技能树、职业、连招和元素伤害。
- Boss、任务、剧情章节、NPC 和对话系统。
- 昼夜、天气、复杂视线遮挡和声音传播模拟。
- 疯狂的实质负面效果，例如幻觉目标、错误信息和操作干扰。
- 多类型尸体资源，例如骨、皮、器官和污染组织。
- 正式美术资源、正式音乐、完整音效库和本地化系统。

## 6. Deprecated / Superseded

- 户外实时自由移动与即时战斗：已被 20×20 格子探索和接敌后独立回合制战斗取代。
- 玩家固定先手：已被速度决定战斗初始先后手取代。
- 敌人从发现直接追击：已加入 `Alert` 与 `AttackIntent` 阶段。
- 战斗主菜单直接显示“吃怪物肉”：已改为“道具 → 异变肉块”。
- 依赖 DOM 按钮焦点的战斗键盘监听：已改为接入游戏运行时统一输入。
- 战斗、切割、等待和撤离消耗饥饿：已废弃，只有户外地图移动扣除饥饿。
- 户外初始饥饿 82：已废弃，默认值为 100。
- 旧状态名 `Attack`：当前地图危险阶段使用 `AttackIntent`，真正攻击在独立战斗中结算。

## 7. 下一阶段边界

### 7.1 允许优先调整

- 当前核心循环的节奏、数值、反馈、操作和可读性。
- 庇护所食物与跨外出生命/饥饿状态的关系。
- 道具与背包从“只有异变肉”扩展到少量明确用途的资源。
- 现有三类怪物的参数、表现和小规模行为差异。
- 随机事件内容、疯狂阶段反馈和风险收益选择。
- Demo 目标、失败压力、撤离决策和单局时长。
- 配置校验、编辑体验、测试覆盖与移动端体验。

### 7.2 当前阶段明确不做

- 实时联机或大型服务端系统。
- 枪械、复杂装备构筑、制作交易和经济循环。
- 多地图、程序化大地图或完整开放世界。
- Boss、职业、技能树和大型任务系统。
- 在没有重新确认版本范围前，不启动 Reserved 或 Not Implemented 中的大型系统。

如果新需求触及上述边界，应先形成新的版本增量文档，再进入实现。

## 8. 文档与开发工作流

```text
提出新需求
→ 先检查 current-game-status.md
→ 判断是新增、调整还是已有功能
→ 更新对应设计增量文档
→ Codex 实现并补充测试
→ 验收后同步更新 current-game-status.md
→ 自动合并与部署
```

需求分析时必须明确区分：

- `Implemented`：当前代码和线上版本可以实际体验。
- `Partially Implemented`：已有可见或底层实现，但尚未形成完整玩法。
- `Reserved`：只有配置字段、数据结构或接口预留。
- `Not Implemented`：代码与玩法均未实现。
- `Deprecated / Superseded`：历史设计已被新规则覆盖，不得按旧文档恢复。
