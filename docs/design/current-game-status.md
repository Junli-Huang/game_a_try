- 当前版本：V1.3.9 Simple Fog Mask
- 状态依据：`main` 分支实际代码、自动测试与本地生产构建。
- 在线状态：公开入口已核对可访问，但线上版本可能短暂落后于 `main`，具体以各部署平台当次构建为准。

## 1. 当前核心循环

```text
主菜单
→ 庇护所查看目标、库存、装备、圣遗物和种植
→ 进入可配置的 10×10～50×50 户外地图
→ 探索迷雾、处理随机事件、观察敌人意图
→ 接敌并进入独立回合制战斗
→ 击杀后切割尸体获得异变肉
→ 环境污染持续消耗疯狂抗性，选择继续深入、食用异变肉或前往撤离点
→ 成功撤离后将本次异变肉带回庇护所
→ 使用圣遗物恢复抗性或净化异变肉
→ 推进种植、Demo 目标与胜负结算
```

默认 Demo 胜利条件：成功撤离 3 次，并累计带回至少 12 块异变肉。

默认失败条件：外出失败达到 3 次，或庇护所的储备粮与异变肉同时耗尽。

## 2. Implemented

### 2.1 主菜单、存档与结算

- 开始游戏、继续游戏、配置、关于、重置存档。
- 存档使用浏览器 `localStorage`，保存生命、疯狂、疯狂抗性、圣遗物、逐块异变肉污染、种植、外出次数、目标进度和统计数据。
- 旧存档通过迁移升级；旧整数肉量会在庇护所库存和户外快照中分别转换为等量满污染肉块，不丢失数量。
- 首次目标说明、胜利/失败结算、继续游戏与重新开始。
- 九个事件触发式首次引导：Demo 目标、户外移动、战争迷雾、敌人警觉、首次战斗、首次切割、异变肉、巢穴和撤离；支持跳过全部、保存完成状态及在配置页重置，出现时暂停游戏输入。
- 成功撤离或失败后展示单次探索回合、探索格数、击杀、破坏巢穴、获得/食用异变肉与疯狂变化；失败原因区分战斗、饥饿和其他运行时原因。
- 最终胜利结算展示总外出、成功撤离、失败、击杀、破坏巢穴、带回异变肉和最高疯狂；最终失败按失败上限或食物耗尽显示不同文案。

### 2.2 庇护所

- 查看生命、疯狂抗性、疯狂阶段、固定默认装备、食物库存、最近一次外出结果和 Demo 目标。
- 食用储备粮或异变肉；异变肉增加的疯狂取该肉块当前剩余污染，不被玩家抗性抵消，并优先食用污染最低的一块。
- 庇护所内有可见、可交互的静默圣遗物，显示当前/最大净化值、玩家抗性、庇护屏障状态和预计消耗；恢复抗性与净化肉各有独立消耗倍率，默认均为 1:1。净化时由玩家从污染肉列表中选择一块确认，可完全或部分净化；抗性已满、没有受污染的肉或可处理量为零时按钮禁用，不产生负数。
- 一个灰麦种植格：消耗储备粮播种，按外出次数生长，成熟后产出储备粮。
- 从庇护所进入唯一的户外区域。

### 2.3 户外地图与战争迷雾

- 地图宽高支持 10～50 格；默认仍为 20×20，包含一个出生点、两个撤离点、障碍、固定/随机怪物点和固定 Seed。
- 大地图运行时显示玩家附近最多 20×20 格的跟随视口，不通过缩小格子强行显示全图。
- 方向键或 WASD 移动，也可以点击相邻格；手机提供屏幕方向键。
- 三态战争迷雾：未探索、已探索记忆、当前可见。
- 当前可见区域保持正常亮度；离开视野后立即切换为低亮度记忆区。默认已探索亮度由 0.38 调整为 0.18，旧默认配置会自动迁移，自定义亮度仍保留。
- 玩家迷雾与敌人视野共用格线遮挡函数；玩家迷雾在地图开启 `terrainBlocksVision` 时应用遮挡，敌人方向视野始终受障碍遮挡。障碍格本身可见，后方不可见。
- 离开视野后保留最后观察到的敌人与尸体记忆，不继续刷新记忆画面。
- 未探索区使用简单的实体黑色遮罩，不再绘制雾团切口、柔化雾沿或模糊边界；该调整只改变遮罩表现，不改变实际可见格、遮挡或探索判定。
- 地图采用离散回合：玩家移动、等待、切割或撤离读条会推进敌人行动。
- 每张地图可配置实时环境污染；默认每 5 秒施加 0.1 点，先消耗疯狂抗性，抗性不足时仅将剩余污染加入疯狂。引导暂停、战斗、转场、地图事件及页面不可见期间不计时；运行中的一次更新跨过多个间隔会补齐结算。抗性耗尽提示在单次外出中只显示一次。

### 2.4 敌人与危险反馈

- 四套数据预设：静默菌兽、游荡者、寻迹者、腐化巢穴。
- 共用统一参数化状态机：`Idle / Wander / Alert / Chase / AttackIntent / Return / Cooldown / Dead`。
- 普通敌人拥有独立四方向朝向和可配置的扇形视野；只有进入视野距离、夹角且未被障碍遮挡的玩家才会被发现。
- 支持基于朝向、扇形范围和障碍遮挡的感知，以及警觉延迟、追踪、攻击意图、离家限制、返回和脱战冷却。
- 敌人会在行动阶段内按状态转向观察；转向不消耗移动、不额外推进地图回合，也不会推进饥饿、切割、撤离或环境污染结算，因此不能借转向获得额外移动或攻击。
- 当前可见普通敌人的视野以连续弧形扇面显示，并以贴近本体、随方向旋转的三角指针标记朝向；扇面由现有格线视野结果裁剪，仍受障碍遮挡。Alert、追踪/攻击意图和冷却复用现有危险色系。迷雾与记忆状态不显示或更新敌人的实时朝向和视野，死亡敌人和巢穴不显示。
- 相邻但位于主动敌人背面的玩家不会仅因距离被发现；玩家主动尝试进入敌人所在格仍会接敌。
- 敌人发现、追踪和即将接敌时提供文字、颜色与危险提示。
- 腐化巢穴仍使用统一怪物实体和战斗/尸体/切割流程；它不移动、不主动接敌，按配置回合产出子怪。地图上有独立轮廓；首次进入视野、产出与破坏分别提供“发现……仍在蠕动”“巢穴中爬出了新的生物”“巢穴停止了蠕动”反馈。首次发现提示不受 UI 开关抑制；产出提示受“敌人警觉提示”UI 开关控制。
- 巢穴遵守首次延迟、产出间隔、存活子怪上限、总产出上限和产出半径；死亡后停止产出，已有子怪继续存在。
- 子怪记录所属巢穴，并可使用巢穴位置作为 Home 中心。
- 玩家通过尝试移动进入巢穴所在格主动与巢穴接敌并进入普通战斗；巢穴自身不会主动触发接敌。

### 2.5 战斗

- 接敌冻结地图，播放敌人特写转场，再进入独立战斗画面。
- 显示双方生命、攻击、速度、疯狂、回合与战斗日志。
- 速度较高的一方在战斗开始时先行动。
- 玩家行动：攻击、防御、道具、逃跑。
- “道具”进入二级菜单；当前仅包含本次外出获得的异变肉。
- 战斗键盘输入接入游戏运行时：方向键或 WASD 移动选项，回车或空格确认。
- 当前选项显示金色高亮；鼠标和触屏仍可直接操作。
- 胜利留下尸体；失败返回庇护所；逃跑使敌人暂时进入冷却状态。
- 玩家和怪物在战斗中实际损失的 HP 由当前场景按 1:1 吸收为 `sceneMadness`；只计算实际损失，不计超过剩余 HP 的过量伤害。本次场景累计值写入外出快照和探索结算，外出结束后随场景实例清除。

### 2.6 饥饿、疯狂与道具

- 生命作为跨外出状态持久化：成功撤离时保存剩余生命，返回庇护所不会自动回满；下一次外出沿用该生命值。外出失败属于死亡结算，生命恢复为玩家配置的初始值，避免以 0 生命进入下一局。
- 玩家拥有当前/最大疯狂抗性，新游戏默认 10/10；进入或离开户外、成功撤离和失败返回均不会自动恢复，只有庇护所圣遗物可以恢复。
- 饥饿仍在每次进入户外时按玩家配置初始化；默认饥饿为 100/100。
- 只有在户外地图实际移动才扣除饥饿，默认每格消耗 1。
- 战斗、等待、切割、撤离读条、事件推进和使用道具不扣除饥饿。
- 饥饿归零后，后续移动会受到饥饿伤害。
- 户外“使用道具”入口和战斗“道具”菜单均可食用异变肉。
- 每块异变肉保存 `currentMadness` 与 `maxMadness`，新生成肉默认 12/12；净化会降低当前值，食用只增加当前剩余值。异变肉默认恢复 10 点生命和 26 点饥饿；生命恢复不会超过上限，户外、战斗与庇护所使用同一污染规则；疯狂阶段提高攻击倍率。
- 疯狂阶段：清醒、低语、狂热、失控边缘；包含阶段文案、低语、边缘效果和减少动态效果选项。

### 2.7 尸体、切割与撤离

- 怪物死亡后在地图对应格留下尸体。
- 站在尸体格可点击高亮按钮或按 E 切割。
- 切割按怪物配置推进多个地图回合，并受背包容量、装备和切割倍率影响。
- 当前尸体资源只有异变肉；切割生成独立肉块实例，当前疯狂初始等于配置的最大疯狂。
- 站在撤离点可点击高亮按钮或按 E，完成配置回合数后撤离。
- 成功撤离保留本次每块肉及其污染状态；失败默认丢失本次肉，固定装备保留，疯狂按配置保留，疯狂抗性始终保留。

### 2.8 随机事件

- 8 个数据驱动事件：废弃背包、未熄灭的火堆、残缺笔记、腐烂储藏箱、远处脚步、陌生尸体、雾中呼唤、熟悉味道。
- 支持启用开关、权重、疯狂条件、事件间隔、单次外出上限、选择和随机结果。
- 事件可以恢复生命/饥饿、增加疯狂、给予食物或推进地图回合。
- 负数饥饿事件效果不生效，饥饿扣除仍只来自地图移动。

### 2.9 配置系统

- 分类：全局、玩家、怪物、食物、怪物肉、圣遗物、疯狂、装备、地图数据、地图编辑、战斗、UI、Demo 目标、疯狂表现、地图事件、音频、种植、庇护所。
- 支持表单编辑、本地保存、应用、恢复默认、基础校验、JSON 导入与导出。
- 地图编辑器支持尺寸调整、可滚动画布、选择/地面/障碍/出生点/撤离点/固定怪物/删除工具、矩形随机怪物区域、对象属性面板、范围叠层、50 步撤销/重做、单张地图 JSON 导入导出、仅恢复默认地图，以及未保存草稿试玩。地面和删除工具可移除地形、固定怪物与非最后一个撤离点；“清空对象”清空障碍、固定怪物和随机区域，保留出生点与撤离点。
- 支持固定 Seed、生成/复制 Seed，以及由矩形区域、数量、距离和最大尝试次数驱动的随机怪物布点。随机布点避开障碍、出生点、撤离点、固定怪物配置坐标和已有随机点，并对固定点和已有随机点应用“任意怪物”与“同类型”距离规则。
- 当地图空间不足、无法达到随机规则目标数量时，系统在最大尝试次数后结束生成，保留已成功放置的实体，不阻塞游戏启动，也不会进入无限循环。
- 怪物配置支持新增、复制任意项、删除无引用的任一非内置怪物及恢复内置怪物配置；巢穴的 `spawnConfig` 使用通用嵌套表单编辑。
- 配置在下一次开始游戏/进入流程时加载，不支持运行中完整热更新。
- 旧战斗行动 `eat` 自动迁移为 `item`；旧地图配置自动补充撤离点列表、随机配置和环境污染配置，缺少内置巢穴时会补入该怪物配置；旧配置同时补入玩家抗性、怪物肉与圣遗物字段。

### 2.10 当前外出恢复

- 户外探索状态保存地图 ID、Seed、伪随机生成器状态、回合、玩家生命/疯狂/抗性、逐块肉污染、环境污染间隔进度、场景疯狂、怪物朝向/最近移动/最后目击位置、尸体、巢穴产出进度、已访问格、已发现巢穴、地图事件服务状态和迷雾记忆。旧快照缺少朝向时根据最近移动或 Seed、敌人 ID 与位置生成稳定方向，不会在每次刷新时随机改变。
- 刷新页面后继续已保存的户外探索状态时，不重新随机现有怪物点位或重置巢穴计时，也不会改变后续依赖运行时随机数的结果；环境污染从已保存的间隔进度继续，但页面关闭或不可见期间不推进污染。
- 地图事件服务的已触发事件、计数和间隔会恢复；战斗、战斗转场和事件选择框本身不在快照中。在这些界面刷新会回到最近一次已保存的户外探索状态，而不是精确恢复界面中间态。
- 正常撤离或失败结算后清除外出快照。

### 2.11 展示与部署

- 桌面与移动端响应式布局，移动端地图保持完整正方形并考虑安全区。
- 页面全局显示低干扰版本号，主菜单、庇护所、户外、战斗与配置界面均可直接确认当前构建版本。
- 户外地图保留格子规则但弱化棋盘感：格线为极淡虚线，相邻障碍不绘制内部共享线；地面带稳定的低对比微纹理。未探索区使用无额外边界特效的实体黑色遮罩。
- 22 个 44.1 kHz / 16-bit / Mono 的程序化 WAV 音效覆盖移动、战斗、敌人状态、切割、物品、污染、圣遗物和撤离；首次交互解锁后加载缓存，单文件失败使用低音量合成回退。
- 自动测试覆盖配置、迷雾记忆、战斗、敌人意图、道具、饥饿、目标、疯狂、事件、地图尺寸、Seed 随机布点、巢穴产出、引导状态迁移与单次外出统计。V1.3.2 专项覆盖抗性、圣遗物、污染、净化、暂停计时、存档迁移与战斗 HP 向场景疯狂的转换；V1.3.3 专项覆盖方向视野、障碍遮挡、行动与快照恢复；V1.3.4 专项覆盖 WAV 规格、固定 Seed、清单一致性、音量、变体、冷却、加载回退与资源释放；V1.3.5 专项覆盖视觉状态色、方向旋转、稳定地表扰动与障碍内部格线抑制。尚未覆盖编辑器 DOM 交互、`requireWalkableTile` 开关或战斗/事件界面中刷新恢复。
- 构建脚本会产出 GitHub Pages 所需的静态文件；GitHub Pages 与 chatgpt.site 的公开入口已完成访问核对。具体发布版本仍以各部署平台当次构建状态为准。

## 3. Partially Implemented

### 3.1 庇护所恢复与食物

- 庇护所有食用按钮和库存扣除逻辑。
- 庇护所显示当前生命；食物按统一配置恢复生命，异变肉同时增加疯狂。成功撤离保留的生命变化会延续到下一次外出。
- 饥饿尚未作为跨外出状态持久化，每次外出仍按玩家配置重新初始化。
- 因此“在庇护所通过食物恢复下一次外出的饥饿”尚未形成完整玩法，储备粮目前主要用于失败条件和种植投入。

### 3.2 装备

- 装备数据、默认装备、攻击、防御和切割产量加成已经接入。
- 当前只能使用配置中标记为默认装备的固定组合。
- 没有游戏内换装、装备库存、掉落、耐久、词条或成长。

### 3.3 道具与背包

- 有背包容量和道具入口，但运行时背包当前只存储逐块记录污染值的异变肉。
- 储备粮配置声明可在户外食用，但目前不会被带入户外背包。
- 没有通用道具实例、堆叠管理、丢弃、排序或多类型搜刮。

### 3.4 音频

- 程序化 WAV 短音效、合成回退、静音、主音量和音效音量已经实现。
- 音效地址使用 Vite `BASE_URL`，兼容 GitHub Pages 的 `/game_a_try/` 子路径。
- 浏览器不会在页面加载时强制播放；首次指针或键盘交互负责解锁音频上下文。当前没有 BGM。

### 3.5 高级内容编辑体验

- 地图编辑器尚不支持图层、多对象选择、框选、对象批量操作或更高效的移动端编辑。
- 地图草稿试玩在当前页面会话中使用临时配置和独立运行状态，不覆盖正式配置和正常游戏存档；通过“返回地图编辑”退出或试玩结算后会恢复原配置与存档，并返回地图编辑页面继续修改。但试玩上下文只保存在内存中，试玩期间刷新页面后不能恢复到编辑页面。
- 编辑器 DOM 交互尚无自动化测试；巢穴参数仍通过通用嵌套表单修改，装备与事件仍没有与怪物配置同等级的专用内容管理操作。

### 3.6 默认节奏与移动端实机验收

- 默认地图使用固定 Seed、20×20 尺寸和两个撤离点，已具备安全起步与远端巢穴风险区域的基础结构。
- 代码和响应式样式已覆盖桌面与移动端核心交互，屏幕方向键禁止默认触控滚动。
- “首次事件 30～90 秒、首次战斗 1～3 分钟、单次外出 10～15 分钟”仍需按 `docs/testing/v1.3.1-mvp-smoke-checklist.md` 完成人工试玩记录；在完成前不视为已验证的体验指标。

## 4. Reserved

- `enableOutdoorFarming`、作物 `allowOutdoor`：为户外种植预留，默认关闭。
- 怪物 `carriedLoot`：为怪物携带物掉落表预留，当前为空。
- 多件装备配置：为后续装备选择预留，当前只读取默认装备。
- `AudioService.playBgm/stopBgm`：为背景音乐预留。
- 事件的条件、权重和永久记录字段：可用于继续增加事件内容。
- 配置版本、云端配置、多套配置方案：架构可扩展，但当前只使用本地存储。
- 胜利后的经验、特殊资源和成长展示位置：只保留表现方向，没有数值系统。
- 地图编辑器的图层、多对象选择、框选、对象批量操作和更高效的移动端编辑：保留为后续体验优化。
- 场景永久化与跨外出污染总账：当前 `sceneMadness` 只在单次外出实例与快照中存在。

Reserved 只表示数据结构或接口已经留位，不代表对应玩法可用。

## 5. Not Implemented

- 服务端、账号、云存档、排行榜、留言或实时联机。
- 多张户外地图、完整程序化地形生成、地图选择和商业级关卡编辑器。
- 户外种植、怪物破坏作物和污染作物生态。
- 装备掉落、换装、耐久、随机词条、制作、交易和经济系统。
- 枪械、弹药、远程战斗、技能树、职业、连招和元素伤害。
- Boss、任务、剧情章节、NPC 和对话系统。
- 昼夜、天气、动态/可破坏遮挡、声音传播和更复杂的感知模拟。
- 疯狂的实质负面效果，例如幻觉目标、错误信息和操作干扰。
- 主动污染怪物肉、圣遗物充能/升级、多圣遗物和庇护所屏障失效。
- 多类型尸体资源，例如骨、皮、器官和污染组织。
- 完整角色/怪物/场景正式美术资源、正式音乐、完整音效库和本地化系统。

## 6. Deprecated / Superseded

- 户外实时自由移动与即时战斗：已被可配置的 10×10～50×50 格子探索（默认 20×20）和接敌后独立回合制战斗取代。
- 玩家固定先手：已被速度决定战斗初始先后手取代。
- 敌人从发现直接追击：已加入 `Alert` 与 `AttackIntent` 阶段。
- 战斗主菜单直接显示“吃怪物肉”：已改为“道具 → 异变肉块”。
- 依赖 DOM 按钮焦点的战斗键盘监听：已改为接入游戏运行时统一输入。
- 战斗、切割、等待和撤离消耗饥饿：已废弃，只有户外地图移动扣除饥饿。
- 户外初始饥饿 82：已废弃，默认值为 100。
- 旧状态名 `Attack`：当前地图危险阶段使用 `AttackIntent`，真正攻击在独立战斗中结算。
- 地图必须固定 20×20：已被 10×10～50×50 配置尺寸与跟随视口取代，默认地图仍保持 20×20。

## 7. 下一阶段边界

### 7.1 允许优先调整

- 当前核心循环的节奏、数值、反馈、操作和可读性。
- 庇护所食物与跨外出生命/饥饿状态的关系。
- 道具与背包从“只有异变肉”扩展到少量明确用途的资源。
- 现有四类怪物的参数、表现、随机布点和巢穴产出节奏。
- 随机事件内容、疯狂阶段反馈和风险收益选择。
- Demo 目标、失败压力、撤离决策和单局时长。
- 配置校验、编辑体验、测试覆盖与移动端体验。

### 7.2 当前阶段明确不做

- 实时联机或大型服务端系统。
- 枪械、复杂装备构筑、制作交易和经济循环。
- 多地图、程序化大地图或完整开放世界。
- Boss、职业、技能树和大型任务系统。
- 在没有重新确认版本范围前，不启动 Reserved 或 Not Implemented 中的大型系统。

如果新需求触及上述边界，应先形成新的版本增量文档，再进入实现。

## 8. 文档与开发工作流

```text
提出新需求
→ 先检查 current-game-status.md
→ 判断是新增、调整还是已有功能
→ 更新对应设计增量文档
→ Codex 实现并补充测试
→ 验收后同步更新 current-game-status.md
→ 自动合并与部署
```

需求分析时必须明确区分：

- `Implemented`：`main` 分支当前代码已经实现并通过相应验证；不表示所有线上入口已同步到同一提交。
- `Partially Implemented`：已有可见或底层实现，但尚未形成完整玩法。
- `Reserved`：只有配置字段、数据结构或接口预留。
- `Not Implemented`：代码与玩法均未实现。
- `Deprecated / Superseded`：历史设计已被新规则覆盖，不得按旧文档恢复。
